Unified Checkout

Request Capture Context

Once you are ready to render the checkout page, you need to request the Capture Context from the /up/v1/capture-contexts endpoint. The capture context request below drives the Click to Pay Drop-in UI's behavior, including accepted card networks, address information to be collected, and more. The capture context request also allows you to optionally pass in previously collected information (e.g. billing and shipping) to pre-fill this into The Click to Pay Drop-in UI. The request produces a JWT that is used to initialize the UI.
